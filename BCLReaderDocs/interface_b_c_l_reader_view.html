Detailed Description

A BCLReaderView object is a view that renders a book and provides built in gesture handling for page turning.

Within an application, only one BCLReaderView should exist at a time, or the results are undefined.

Method Documentation

§ addHighlightForCurrentLocation()

- (nullable BCLHighlight *) addHighlightForCurrentLocation

Adds a highlight based on the current location.

The highlight will have a location with no range, so it will not render with a highlighted effect. The advantage of calling this method over the location property is that highlights have a unique ID within a book, which is useful when persisting highlights for a feature like bookmarks or annotations.

Returns
A BCLHighlight or nil if the highlight was unable to be created.

- (nonnull UISwipeGestureRecognizer *) addSwipeDownGestureWithTarget: (nonnull id)  target
action: (nonnull SEL)  action 

Adds a swipe down gesture recognizer to the view.

The reader view has built in handling for left and right swipes, which are used to turn the page. Due to the way gestures are handled internally, an application should not add gesture recognizers directly to the reader view. Instead, call this method to have a swipe down gesture added.

Parameters
targetThe target to call when a swipe down occurs.
actionThe selector to call when a swipe down occurs.
Returns
A UISwipeGestureRecognizer object.

§ toggleMediaOverlayIsPlaying()

- (void) toggleMediaOverlayIsPlaying

If media overlay is available, changes the playing state to paused, or changes the paused state to playing.

Call isMediaOverlayAvailableWithCompletion: to determine if media overlay is available. The isMediaOverlayPlaying property indicates whether or not media overlay is currently playing. The readerViewMediaOverlayIsPlayingDidChange: delegate method is called whenever the isMediaOverlayPlaying value changes.

§ pageList

- (BCLPageList*) pageList
readnonatomicstrong

The page list.

When a reader view is first created, the page list will be empty. An asynchronous process calculates the pages available in the book. As the calculations make progress, the readerView:pageListInitializationDidMakeProgress: delegate method is called. Once the process has completed, this property will contain the page list.
