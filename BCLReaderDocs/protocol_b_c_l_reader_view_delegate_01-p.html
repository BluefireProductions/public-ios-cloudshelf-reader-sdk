§ readerView:initializeDidFinishWithError:()

- (void BCLReaderViewDelegate) readerView: (nonnull BCLReaderView *)  readerView
initializeDidFinishWithError: (nullable NSError *)  error 
optional

Called when the reader view has finished initializing.

When a reader view is first created, it is not ready for use. It takes some time for the book to be processed. This method is called once initialization has completed. Note that initialization will finish prior to readerView:pageListInitializationDidMakeProgress: being called.

Parameters
readerViewThe reader view.
errorAn error if initialization failed.

§ readerView:pageListInitializationDidMakeProgress:()

- (void BCLReaderViewDelegate) readerView: (nonnull BCLReaderView *)  readerView
pageListInitializationDidMakeProgress: (double)  progress 
optional

Called when page list initialization makes progress.

When a reader view is first created, the page list will be empty. An asynchronous process calculates the pages available in the book. As the calculations make progress, this method is called. The value of progress could be used by an application to render a progress bar, for example.

Parameters
readerViewThe reader view.
progressA value between 0.0 and 1.0 representing the progress of initialization. Once the value reaches 1.0, initialization is complete and the pageList property will have the list of pages.

§ readerViewDidTap:()

- (void BCLReaderViewDelegate) readerViewDidTap: (nonnull BCLReaderView *)  readerView
optional

Called when a tap occurs in the reader view.

Some taps are consumed internally. For example, a tap in the left or right margin will generally cause a page turn. This method is called only when the reader view has not consumed the tap itself. An application might use this event to toggle the visibility of user interface controls, for example.

Parameters
readerViewThe reader view.

§ readerViewHeartbeatDidFail:()

- (void BCLReaderViewDelegate) readerViewHeartbeatDidFail: (nonnull BCLReaderView *)  readerView
optional

Called under rare circumstances when the book becomes unresponsive.

Occasionally, some books become unresponsive on some devices. This method is called in such a case. An application should deallocate the reader view in response, possibly displaying an error message to the user.

Parameters
readerViewThe reader view.
